# ĐỀ 01: XÂY DỰNG WEBSITE + LOGIN THƯƠNG MẠI SỬ DỤNG MARIADB + WORDPRESS  
## ***1. Lý thuyết và Công nghệ sử dụng (Tech Stack)***
- Phần này giúp người xem hiểu bạn đang dùng những gì và tại sao lại dùng chúng.

- Hệ điều hành (OS): Ubuntu 24.04 LTS (Linux) - Cung cấp môi trường máy chủ ổn định, bảo mật và hiệu suất cao cho Docker.

- Docker: Nền tảng ảo hóa dạng container, giúp đóng gói ứng dụng và các thành phần phụ thuộc để chạy nhất quán trên mọi môi trường.

- Docker Compose: Công cụ hỗ trợ định nghĩa và khởi chạy đa container (Multi-container) thông qua một file cấu hình duy nhất (docker-compose.yml).

- WordPress (Frontend/CMS): Hệ quản trị nội dung mã nguồn mở chạy trên nền tảng PHP, đóng vai trò là giao diện và bộ xử lý chính cho website thương mại.

- MariaDB (Database): Một hệ quản trị cơ sở dữ liệu quan hệ (RDBMS) mã nguồn mở, được tối ưu hóa để lưu trữ dữ liệu người dùng, sản phẩm và bài viết cho WordPress.

- Adminer (DB Management): Công cụ quản lý cơ sở dữ liệu siêu nhẹ chạy trên nền PHP, dùng để thao tác trực tiếp với MariaDB thông qua trình duyệt.

## ***2. Cấu trúc dự án (Project Structure)***
Việc phân chia thư mục rõ ràng giúp bạn quản lý mã nguồn và dữ liệu tách biệt, tránh nhầm lẫn khi triển khai thực tế.
```
ecommerce-web/
├── docker-compose.yml       # File cấu hình chính để khởi chạy các dịch vụ.
├── wp-content/              # Thư mục chứa Plugin, Theme và ảnh tải lên (được ánh xạ từ Docker).
│   ├── plugins/             # Nơi bạn sẽ code các Plugin quản lý sinh viên/thương mại.
│   └── themes/              # Giao diện của website.
└── .env                     # (Nâng cao) File chứa mật khẩu và biến môi trường bảo mật.
```
## ***3. Quy trình triển khai Project (Workflow)***
- Khởi tạo: Sử dụng Terminal Ubuntu để tạo không gian làm việc (mkdir, cd).

- Định nghĩa: Viết file docker-compose.yml để thiết lập mối quan hệ giữa WordPress và MariaDB.

- Triển khai: Chạy lệnh docker compose up -d để tải Image và tạo Container.

- Phân quyền (Critical): Sử dụng lệnh chown và chmod trên Linux để cấp quyền ghi cho máy chủ web (www-data) vào thư mục wp-content nhằm tránh lỗi tải file.
